📋 우선순위 1: 거시경제 '회피' 시스템 (Macro Event Filter)
[필요성: 매우 높음] 현재 봇은 차트와 호가만 보고 있어, CPI 발표나 금리 결정 같은 **'외부 충격'**에 무방비 상태입니다. 보고서에서 언급한 대로 이 시간대에는 기술적 분석이 무용지물이 되므로 반드시 피해야 합니다.

❌ 미구현 기능:

경제 캘린더 연동: 주요 경제 지표 발표 시간(미국 CPI, FOMC, 고용지표) 전후 30분간 매매를 **자동 일시 정지(Pause)**하는 기능.

나스닥 개장 필터: 한국 시간 밤 11시 30분(썸머타임 적용 시 10시 30분) 전후의 윔소(Whipsaw) 장세 회피 기능.

📥 더 필요한 데이터:

Economic Calendar Data: Investing.com 크롤링 또는 ForexFactory API 등을 통해 '별 3개(중요도 높음)' 일정의 날짜와 시간 데이터.

📋 우선순위 2: 호가창 '속임수' 판독기 (Advanced Tape Reading)
[필요성: 높음] 현재 order_manager.py에는 단순히 '매수/매도 잔량 비율'만 비교하는 로직(analyze_orderbook_health)만 존재합니다. 보고서의 핵심인 **'허수(Fake Wall)'**와 **'스윕(Sweep)'**을 잡아내려면 데이터 처리 방식이 바뀌어야 합니다.

❌ 미구현 기능:

허수 주문 감지: 호가창 스냅샷을 1초 단위로 비교하여, 체결 없이 사라지는 대량 주문(취소 주문)을 포착하고 해당 방향 진입을 차단하는 로직.

스윕(Sweep) 감지: 누군가 시장가로 매도 1~3호가를 한 번에 긁어가는 **'체결 속도'**를 계산하여 추세에 동참(Momentum Trading)하는 로직.

📥 더 필요한 데이터:

Real-time Trade Ticks (체결 내역): 현재 코드는 ticker(현재가) 정보만 받습니다. 업비트 웹소켓에서 trade 타입을 추가 구독하여 **'초당 체결 건수'**와 '체결 강도' 데이터를 실시간으로 수집해야 합니다.

Orderbook Diff (호가 변경분): 호가 잔량의 단순 합계가 아니라, '직전 호가 대비 증감분' 데이터가 필요합니다.

📋 우선순위 3: 온체인 고래 감시 (Whale Alert)
[필요성: 중간 (특정 코인 한정)] 보고서에서 언급한 '리플(XRP) 입금 감지' 전략을 구현하려면 거래소 밖의 데이터가 필요합니다. 업비트로 대량의 코인이 입금되는 것은 강력한 매도 신호입니다.

❌ 미구현 기능:

거래소 유입 급증 감지: 특정 코인(XRP, ETH 등)이 업비트 지갑으로 대량 이체될 때, 매수를 멈추거나 보유 물량을 매도하는 방어 로직.

📥 더 필요한 데이터:

On-chain Data API: CryptoQuant, Whale Alert, 또는 Arkham Intelligence 같은 외부 API를 통해 'Exchange Inflow(거래소 입금량)' 데이터를 주기적으로(예: 5분마다) 호출해야 합니다.

📋 우선순위 4: VWAP '매수' 로직 정교화
[필요성: 중간] 현재 signal_maker.py에 is_vwap_support가 있지만, 이는 단순히 "현재가가 VWAP보다 위에 있는가?"를 체크하는 필터(Filter) 역할에 가깝습니다.

❌ 미구현 기능:

VWAP 눌림목(Bounce) 트리거: 가격이 위에서 내려오다가 VWAP 라인을 **'터치하고 다시 반등하는 순간'**을 포착하는 구체적인 진입 로직. (현재는 단순히 위에 있으면 OK로 처리됨)
